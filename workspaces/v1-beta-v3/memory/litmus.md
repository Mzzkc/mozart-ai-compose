## Learned Lessons
- Empty tag list `spec_tags: {5: []}` returns ALL fragments, not none. May surprise score authors — design decision, but non-obvious.
- Fragment name collision: `intent.md` and `intent.yaml` both produce fragments named "intent". No dedup. Potential silent data loss.
- CLAUDE.md fragment has `tags=[]`, so it gets excluded when tag filtering is active. Tagless fragments being silently filtered is a trap for score authors.
- Deliberately avoided duplicating existing `test_spec_budget_gating.py` coverage. Always check what exists before writing new tests.
- The baton's rate limit handler correctly refuses to transition PENDING sheets to WAITING (only dispatched/running sheets transition). Dispatch checks instrument state separately via build_dispatch_config().
- F-134 was a one-field typo (max_cost_usd → max_cost_per_job) that silently disabled cost protection. No crash, no type error. The system just stops being intelligent. Exactly what litmus tests exist to catch.
- The `is_exhausted` property has a nuance: a sheet with exhausted retries but remaining completion mode budget is NOT exhausted. Exhaustion means ALL modes are spent.
- The shift from "does the intelligence layer help agents?" to "does the production pipeline actually work?" — F-255 changed the question. Before: test whether features IMPROVE things. After: test whether features EXIST at all.
